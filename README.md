## 🧙‍♂️ The System

| Attribute | Description |
|-----------|-------------|
| **Ranks** | E → D → C → B → A → S → SS → SSS |
| **Skills** | 12 unique skills, each with its own XP bar and level cap |
| **Streaks** | Daily login and quest streaks with visual calendar |
| **Challenges** | Time-bound (30/60/90/365 day) missions |
| **Diet Plans** | Toggle between Veg / Non-Veg meal recommendations |
| **Deficit Calc** | Calculate calories, kg/week loss, target weight projection |

## ⚡ Getting Started

```bash
# Clone the repository
git clone https://github.com/manish005/SoloLeveling.git

# Navigate to the project
cd SoloLeveling

# Install dependencies
npm install

# Create environment file
cp .env.example .env
# Edit .env with your Firebase and Cloudinary credentials

# Start development server
npm run dev

# Build for production
npm run build
```

### 🔐 Environment Variables

```env
VITE_FIREBASE_API_KEY=
VITE_FIREBASE_AUTH_DOMAIN=
VITE_FIREBASE_DATABASE_URL=
VITE_FIREBASE_PROJECT_ID=
VITE_FIREBASE_STORAGE_BUCKET=
VITE_FIREBASE_MESSAGING_SENDER_ID=
VITE_FIREBASE_APP_ID=
VITE_CLOUDINARY_CLOUD_NAME=
VITE_CLOUDINARY_UPLOAD_PRESET=
```

## 🎨 Theme

Solo Leveling supports both **Dark** and **Light** modes, inspired by the signature blue-and-black aesthetic of the *Solo Leveling* manhwa. The UI features:

- ✦ Glassmorphism cards with neon borders
- ✦ Gradient "mana" accents (blue → purple → cyan)
- ✦ Glowing ability graph with SVG filters
- ✦ Rank-colored badges (E → SSS)
- ✦ Smooth framer-motion transitions

## 🧬 Project Structure

```
src/
├── assets/          # Static images & icons
├── components/      # Reusable UI components
│   ├── animations/  # Level-up modal, particles, confetti
│   ├── layout/      # Sidebar, Header
│   └── ui/          # ProgressBar, Select, ThemeToggle, AbilityGraph
├── layouts/         # App & Auth layout shells
├── lib/             # Firebase, Cloudinary, database client
├── pages/           # Route pages (Dashboard, Quests, Workout, etc.)
├── router/          # Route definitions & guards
├── services/        # XP, quest, workout business logic
├── store/           # Zustand state (auth, user, UI)
├── types/           # TypeScript interfaces & constants
└── utils/           # cn() utility
```
